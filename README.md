## 🚀 How to Get a Live Link (GitHub Pages)

### Step 1 — Create the Repository
1. Go to **https://github.com** and sign in
2. Click **"+"** → **"New repository"**
3. Name: `FUTURE_FS_03`
4. Set to **Public**
5. Click **"Create repository"**

### Step 2 — Upload the Files
**Option A — Direct Upload (easiest, no terminal needed):**
1. On your new repo page, click **"uploading an existing file"** (shown in the empty repo)
2. Drag and drop both `index.html` and `README.md`
3. Scroll down → type commit message: `feat: add Ember & Bloom café website`
4. Click **"Commit changes"**

**Option B — Via VS Code Terminal:**
```bash
cd FUTURE_FS_03
git init
git add .
git commit -m "feat: Ember & Bloom cafe website - Task 3"
git remote add origin https://github.com/YOUR_USERNAME/FUTURE_FS_03.git
git branch -M main
git push -u origin main
```

### Step 3 — Enable GitHub Pages (Get Live Link)
1. In your repo, click **"Settings"** tab (top menu)
2. Scroll down the left sidebar → click **"Pages"**
3. Under **"Branch"** → select **`main`** from the dropdown
4. Leave folder as **`/ (root)`**
5. Click **"Save"**
6. Wait **1–2 minutes**
7. Refresh the Settings → Pages page
8. You'll see a green box: **"Your site is live at https://YOUR_USERNAME.github.io/FUTURE_FS_03"**
9. Click the link to see your live website! ✅

> 💡 **Tip:** If it shows a 404 for a few minutes, wait — GitHub Pages takes 1-3 minutes to deploy after the first push.
